# Сервис примерки причесок

Веб-сервис для примерки мужских причесок по фотографии. Пользователь загружает портрет, выбирает вариант стрижки и получает готовое изображение с измененной прической. Лицо, фон и общий ракурс должны оставаться максимально близкими к исходному фото.

Проект рассчитан на простой сценарий: быстрый выбор прически, понятный результат для клиента и удобное управление каталогом через админ-панель.

## Возможности

- загрузка фото пользователя;
- выбор прически из заранее подготовленного каталога;
- сохранение родного цвета волос в основном сценарии;
- вывод одного итогового изображения;
- админ-панель с авторизацией;
- добавление и редактирование причесок;
- загрузка изображений для карточек и референсов;
- хранение ключей и служебных настроек только на сервере.

## Стек

- Node.js 18+
- Express
- Helmet
- express-rate-limit
- cookie-parser
- статический интерфейс в `public/`
- данные каталога в `data/catalog.json`

## Запуск локально

Установите зависимости:

```bash
npm install
```

Создайте локальный файл `.env` и укажите в нем настройки сервера и ключи внешнего сервиса обработки. Файл `.env` не должен попадать в репозиторий.

Запустите проект:

```bash
npm start
```

Главная страница:

```text
http://localhost:3000
```

Админ-панель:

```text
http://localhost:3000/admin.html
```

Перед первым запуском на сервере задайте надежные данные администратора через переменные окружения. После входа пароль можно сменить в админ-панели.

## Настройки окружения

Минимальный набор настроек:

```text
# Порт, на котором запускается Node.js приложение.
# Если перед приложением стоит Nginx, обычно можно оставить 3000.
PORT=3000

# Режим обработки причесок.
# Для текущей версии используется vmodel.
HAIR_ENGINE=vmodel

# Ключ API.market. Нужен для загрузки исходных изображений и совместимых сценариев обработки.
API_MARKET_KEY=your_api_market_key

# Токен сервиса обработки причесок.
# Получается в личном кабинете выбранного внешнего сервиса.
VMODEL_API_TOKEN=your_vmodel_token

# Адрес создания задачи обработки.
# Обычно менять не нужно.
VMODEL_CREATE_URL=https://api.vmodel.ai/api/tasks/v1/create

# Адрес проверки статуса задачи.
# Обычно менять не нужно.
VMODEL_STATUS_URL=https://api.vmodel.ai/api/tasks/v1/get

# Версия модели обработки.
# Значение берется из документации или личного кабинета сервиса.
VMODEL_VERSION=your_model_version

# Локальный демонстрационный режим.
# true - внешние запросы не выполняются, false - используется реальная обработка.
MOCK_API=false

# Интервал проверки статуса задачи в миллисекундах.
# 3000 означает проверку каждые 3 секунды.
POLL_INTERVAL_MS=3000

# Максимальное время ожидания результата в миллисекундах.
# 600000 означает 10 минут.
POLL_TIMEOUT_MS=600000

# Логин первого администратора.
# Используется только при первом запуске, если файл data/admin-auth.json еще не создан.
DEFAULT_ADMIN_LOGIN=...

# Пароль первого администратора.
# Укажите сложный пароль и смените его после первого входа в админ-панель.
DEFAULT_ADMIN_PASSWORD=...
```

Значения ключей и пароль администратора задаются только в локальном `.env`. Не публикуйте `.env`, логи, файл с хэшем пароля администратора и временные изображения.

## Каталог

Основной каталог хранится в:

```text
data/catalog.json
```

Через админ-панель можно изменить:

- название прически;
- короткое описание;
- изображение карточки;
- референс для обработки;
- список цветов, если выбранный режим обработки это поддерживает.

Загруженные изображения сохраняются в:

```text
data/references/
data/previews/
```

Файл `data/admin-auth.json` содержит хэш пароля администратора и исключен из репозитория.

## Развертывание

Рекомендуемая схема:

```text
Домен -> Nginx + HTTPS -> Node.js приложение -> внешний сервис обработки
```

Для рабочего сервера:

- используйте HTTPS;
- храните `.env` только на сервере;
- задайте сложный пароль администратора до первого запуска;
- настройте резервное копирование `data/catalog.json`, `data/references/` и `data/previews/`;
- ограничьте размер загружаемых файлов;
- включите регулярную очистку временных изображений и логов;
- отдавайте итоговые изображения через свой домен, чтобы пользователь не зависел от сторонних ссылок.
